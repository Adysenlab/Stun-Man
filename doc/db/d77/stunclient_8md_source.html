3 # NAME
4 
5 stunclient \- command line app for the STUN protocol
6 
7 # SYNOPSIS
8 
9 **stunclient** [OPTIONS] server [port]
10 
11 #DESCRIPTION
12 
13 stunclient attempts to discover the local host's own external IP address, obtain a
14 port mapping, and optionally discover properties of the Network Address Translator (NAT)
15 between the host and the the server.
16 
17 # OPTIONS
18 
19 The following options are supported.
20 
21  --mode MODE
22  --localaddr INTERFACE
23  --localport PORTNUMBER
24  --family IPVERSION
25  --protocol PROTO
26  --verbosity LOGLEVEL
27  --help
28 
29 Details of each option and parameters are as follows.
30 
31 **server**
32 
33 The **server** parameter is the IP address or FQDN of the remote server to perform the binding tests with. It is the only required parameter.
34 
35 _____
36 
37 **port**
38 
39 The **port** parameter is an optional parameter that can follow the server parameter. The default is 3478 for UDP and TCP.
40 
41 _____
42 
43 
44 **--mode** MODE
45 
46 Where MODE is either "basic", "behavior", or "filtering".
47 
48 "basic" mode is the default and indicates that the
49 client should perform a STUN binding test only.
50 
51 "behavior" mode indicates that the client should
52 attempt to diagnose NAT behavior and port mapping methodologies if the server supports this mode.
53 
54 "filtering" mode indicates that the client should
55 attempt to diagnose NAT filtering methodologies if the server supports this mode. The NAT filtering test is only supported for UDP.
56 
57 "full" mode is a deprecated mode. It performs both a filtering and a behavior test together. Users
58 are encouraged to run these tests separately and to avoid using the same local port.
59 
60 ____
61 
62 
63 **--localaddr** INTERFACE or IPADDRESS
64 
65 The value for this option may the name of an interface (such as "eth0" or "lo"). Or it may be
66 one of the available IP addresses assigned to a network interface present on the host (such as
67 "128.23.45.67"). The interface chosen will be the preferred address for sending and receiving
68 responses with the remote server. The default is to let the system decide which address to send
69 on and to listen for responses on all addresses (INADDR_ANY).
70 
71 ____
72 
73 
74 **--localport** PORTNUM
75 
76 PORTNUM is a value between 1 to 65535. This is the UDP or TCP port that the primary and
77 alternate interfaces listen on as the primary port for binding requests. If not specified, a
78 randomly available port chosen by the system is used.
79 
80 ____
81 
82 **--family** IPVERSION
83 
84 IPVERSION is either "4" or "6" to specify the usage of IPV4 or IPV6. If not specified, the default value is "4".
85 
86 ____
87 
88 **--protocol** PROTO
89 
90 PROTO is either "udp" or "tcp". "udp" is the default if this parameter is not specified
91 
92 ____
93 
94 **--verbosity** LOGLEVEL
95 
96 Sets the verbosity of the logging level. 0 is the default (minimal output and logging). 1 shows slightly more. 2 and higher shows even more.
97 
98 ____
99 
100 **--help**
101 Prints this help page
102 
103 # EXAMPLES
104 
105 stunclient stunserver.org 3478
106 : Performs a simple binding test request with the server listening at "stunserver.org"
107 
108 stunclient --mode filtering --localport 9999 12.34.56.78
109 : Performs the NAT filtering tests from local port 9999 to the server listening at IP
110  Address 12.34.56.78 (port 3478)
111 
112 stunclient --mode behavior 12.34.56.78
113 : Performs the NAT behavior and port mapping tests from a random local port to the server listening at IP
114  Address 12.34.56.78 (port 3478)
115 
116 stunclient --protocol tcp stun.selbie.com
117 : Performs a simple binding test using TCP to server listening on the default port of 3478 at
118  stun.selbie.com
